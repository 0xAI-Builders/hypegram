💾 What is Hypegram?

Every time your CI/CD pipeline finishes, someone somewhere is looking at a boring text notification. That someone should be looking at a random dancing cat instead.

Hypegram is a GitHub Action that posts a random GIF to Telegram whenever your workflow runs. Success? Party GIF. Failure? Sad-cat GIF. Every deploy becomes a tiny dopamine hit. It's a drop-in replacement for appleboy/telegram-action — same shape, more vibes.

★ FEATURES ★
  • 🎬 Random GIF per run from a curated pool
  • 🎨 Bring your own GIF list, or use the built-in one
  • 😢 Sad-cat pool for failures (auto-swapped via status)
  • 🪂 Never silent — falls back to plain text if the GIF can't be downloaded
  • 🐚 Pure bash + curl — no Docker, no Node, ~2 seconds per step
  • 🔌 Works in ANY GitHub Actions workflow

🚀 Quick Start

Add these steps to any workflow file (.github/workflows/*.yml):

- name: Notify Telegram on success
  if: success()
  uses: 0xAI-Builders/hypegram@v1
  with:
    token: ${{ secrets.TELEGRAM_BOT_TOKEN }}
    chat_id: ${{ secrets.TELEGRAM_CHAT_ID }}
    caption: |
      ✅ Deploy: ${{ github.repository }}
      📝 ${{ github.event.head_commit.message }}
      👤 ${{ github.actor }}

- name: Notify Telegram on failure
  if: failure()
  uses: 0xAI-Builders/hypegram@v1
  with:
    token: ${{ secrets.TELEGRAM_BOT_TOKEN }}
    chat_id: ${{ secrets.TELEGRAM_CHAT_ID }}
    status: failure
    caption: |
      ❌ Deploy failed: ${{ github.repository }}
      🔗 https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }}

That's it. Ship it.

📥 Inputs

NameRequiredDefaultDescription
token ✅ yes — Telegram bot token. Get one from @BotFather.
chat_id ✅ yes — Target chat or user ID. Send /start to your bot then hit https://api.telegram.org/bot<TOKEN>/getUpdates.
status no success success or failure. Picks the built-in GIF pool that fits the mood.
caption no (empty) Multi-line caption sent with the GIF. Plain text — Telegram won't render Markdown here.
gifs no (built-in) Newline-separated list of GIF URLs. Overrides the built-in pool.

🌍 Use it EVERYWHERE

You probably want one bot + one chat notifying every CI you have. Don't copy-paste secrets across 50 repos like an animal — use organization secrets.

🥇 Option A — Organization Secrets (recommended)

  1. Go to github.com/organizations/<your-org>/settings/secrets/actions
  2. Create TELEGRAM_BOT_TOKEN and TELEGRAM_CHAT_ID
  3. Set the visibility to All repositories (or a specific subset)
  4. Reference them in every workflow as ${{ secrets.TELEGRAM_BOT_TOKEN }}

Now the same YAML snippet works everywhere without duplicating anything.

🥈 Option B — Per-repo secrets

For personal repos or one-offs: Repo → Settings → Secrets and variables → Actions → New repository secret.

"hypegram works in ANY repo — public, private, personal, org — as long as it's a GitHub Actions workflow."

🎨 Custom GIF Pools

Curate your own vibe. The built-in pools are cool, but yours will be cooler.

- uses: 0xAI-Builders/hypegram@v1
  with:
    token: ${{ secrets.TELEGRAM_BOT_TOKEN }}
    chat_id: ${{ secrets.TELEGRAM_CHAT_ID }}
    caption: "Shipped 🚀"
    gifs: |
      https://media.giphy.com/media/xxx/giphy.gif
      https://media.giphy.com/media/yyy/giphy.gif
      https://cataas.com/cat/gif/says/Ship%20it

Any URL that resolves to a GIF (or MP4/MPEG-4) works — Giphy, Tenor, Cataas, your own S3 bucket, whatever.

🔬 How it works

  1. Picks a random URL from the pool (custom gifs if provided, otherwise built-in pool matching status).
  2. Downloads the GIF with curl.
  3. Posts it to Telegram via sendAnimation.
  4. If the download fails, falls back to sendMessage with the caption as plain text.

Steps run in ~2 seconds. No Docker pull, no Node install. Fastest step in your workflow.

❓ Frequently Asked Questions

Can I use Markdown in the caption?

Not yet — captions are plain text. Open an issue if you want it.

Can I send video instead of GIF?

Telegram's sendAnimation accepts GIF, MPEG-4 without sound, and short MP4s. Any of those in the gifs input will work.

What runners does it need?

Anything with bash + curl. Any ubuntu-* image works out of the box. macOS runners too.

Does it work on self-hosted runners?

Yes — as long as the runner can curl both api.telegram.org and the GIF URLs.

Can I use multiple chats?

Add multiple steps, each with a different chat_id. Or use a Telegram group chat and add the bot to it.

Where do I get a chat_id?

  1. Create a bot with @BotFather. Get the token.
  2. Send /start to your bot from your Telegram account (or add it to a group).
  3. Open https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates in a browser.
  4. Find "chat":{"id": ...} in the JSON. That's your chat_id.
