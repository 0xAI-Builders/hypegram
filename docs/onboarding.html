~ ONBOARDING GUIDE ~

★ Get your team hooked up to Telegram deploy notifications with GIFS ★

We use hypegram — a GitHub Action that sends a random GIF to Telegram whenever your CI runs. Success? Party GIF. Failure? Sad-cat GIF. It's like appleboy/telegram-action but with more vibe.

★ WHAT YOU'LL NEED ★
  1. A Telegram bot (2 min)
  2. Your chat_id (1 min)
  3. Two secrets in your repo (1 min)
  4. A YAML snippet in your workflow (30 sec)

Total: ~5 minutes.

1 Create the Telegram Bot

  1. Open @BotFather in Telegram.
  2. Send it /newbot and follow the prompts (pick a name and username).
  3. BotFather gives you a token that looks like:
1234567890:ABCdefGHIjklMNOpqrsTUVwxyz-EXAMPLE-NOT-REAL

Save it. This is your TELEGRAM_BOT_TOKEN.

2 Get Your chat_id

  1. Open a chat with your new bot and send it /start.
    (Or add the bot to a group — group notifications work too.)
  2. Open this URL in a browser (replace with your token):
https://api.telegram.org/bot<YOUR_TOKEN>/getUpdates
  1. Look for "chat":{"id": XXXXX} in the JSON. That number is your TELEGRAM_CHAT_ID.
💡 TIP: Group chat IDs start with -100... — that's normal, use the whole thing.

3 Save Secrets in Your Repo

From your terminal (needs the gh CLI installed & logged in):

gh secret set TELEGRAM_BOT_TOKEN \
  --repo <owner>/<repo> \
  --body '1234567890:ABCdefGHI...'

gh secret set TELEGRAM_CHAT_ID \
  --repo <owner>/<repo> \
  --body '123456789'

Or via the GitHub UI: Repo → Settings → Secrets and variables → Actions → New repository secret.

⚠️ Heads up: If you're in an organization and try to use org-level secrets with visibility "all repositories", some org setups block secret propagation and hypegram will fail with 404 Not Found. Set them as repo-level secrets to be safe.

4 Paste in Your Workflow

Open your workflow file (something like .github/workflows/deploy.yml) and add these two steps at the end of your deploy job:

      - name: Telegram Notification on Success
        if: success()
        uses: 0xAI-Builders/hypegram@v1
        with:
          token: ${{ secrets.TELEGRAM_BOT_TOKEN }}
          chat_id: ${{ secrets.TELEGRAM_CHAT_ID }}
          caption: |
            ✅ Deploy: ${{ github.repository }}
            📝 ${{ github.event.head_commit.message }}
            👤 ${{ github.actor }}
            🔗 https://github.com/${{ github.repository }}/commit/${{ github.sha }}

      - name: Telegram Notification on Failure
        if: failure()
        uses: 0xAI-Builders/hypegram@v1
        with:
          token: ${{ secrets.TELEGRAM_BOT_TOKEN }}
          chat_id: ${{ secrets.TELEGRAM_CHAT_ID }}
          status: failure
          caption: |
            ❌ Deploy failed: ${{ github.repository }}
            🔗 https://github.com/${{ github.repository }}/actions/runs/${{ github.run_id }}

★ DONE. ★

Push a commit that triggers your workflow — a GIF should land in your Telegram chat.

💥 Cool Extras

Custom GIFs

Pass your own vibe with the gifs: input (one URL per line):

with:
  token: ${{ secrets.TELEGRAM_BOT_TOKEN }}
  chat_id: ${{ secrets.TELEGRAM_CHAT_ID }}
  caption: "Shipped 🚀"
  gifs: |
    https://media.giphy.com/media/xxx/giphy.gif
    https://media.giphy.com/media/yyy/giphy.gif
    https://cataas.com/cat/gif/says/Ship%20it

Non-CI use cases

hypegram works in any workflow — not just deploys.

  • 🕔 Scheduled cheer (Friday cron)
  • 🚀 Release announcements (on: release)
  • 🎉 Manual "what are we celebrating?" (workflow_dispatch)

→ See full examples on the home page

Multiple chats

Repeat the step with different chat_id values. Or set the bot to a group and use the group's chat_id — everyone in the group gets notified at once.

🧯 Troubleshooting

ErrorCauseFix
404 Not Found Token secret is empty or malformed Recheck the secret; try as repo secret instead of org secret
403 Forbidden Bot wasn't started or isn't in the group Send /start to the bot or add it to the group
400 chat not found Wrong chat_id Recheck with getUpdates
GIF didn't land but log says "sent" Bot muted in your Telegram settings Unmute the chat
